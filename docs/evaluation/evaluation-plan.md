# RAG 评估计划

## 目标

使用固定、可公开提交的问题集验证检索、引用和回答质量，避免只凭单次演示判断效果。MVP 先记录可重复的基线，不接入外部评估平台。

## 评估资产

- `evals/fixtures/documents/`：可公开的固定测试文档。
- `evals/datasets/`：问题、参考答案、期望引用和标签。
- `evals/configs/`：模型、切分、topK、阈值和提示词版本。
- `evals/reports/`：可选的脱敏基线报告。

## 最小用例类型

- 文档中存在直接答案的问题。
- 需要组合多个分段的问题。
- 文档没有答案、应明确拒答的问题。
- 容易命中相似但错误分段的问题。
- 带页码 PDF 的引用定位问题。
- 模型或 embedding 调用失败时的错误处理。

## 指标

- 检索命中率：期望分段是否进入 topK。
- 引用正确率：引用是否支持对应回答。
- 回答忠实度：回答是否受检索内容支持。
- 拒答正确性：无依据时是否避免编造。
- 延迟和 Token：记录端到端耗时、首字时间和 Token 用量。
- 稳定性：固定配置多次运行的失败率和结果波动。

任何提示词、切分策略、检索参数、默认模型或引用格式变化，都应更新配置并重新生成基线。真实用户文档、密钥和未脱敏请求不得进入评估资产。
