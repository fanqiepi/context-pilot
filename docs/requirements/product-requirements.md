# ContextPilot MVP 产品需求

> 状态：已确认的 MVP 基线

## 产品目标

ContextPilot 是一个面向学习和作品展示的单用户知识库与任务助手。MVP 的目标是完成从资料上传、向量检索到带引用流式回答的最小闭环，并保留可观察、可评估的工程资产。

## 核心流程

1. 创建、查看和删除知识库。
2. 上传 TXT、Markdown 或文本型 PDF。
3. 后台解析、切分、向量化并记录处理状态。
4. 用户提问后检索相关分段，调用大模型生成回答。
5. 通过 SSE 返回增量内容和引用，保存会话、调用记录与反馈。

## MVP 功能

- 文档上传、状态查询、失败提示和有限重试。
- DashScope `text-embedding-v4`，固定 1024 维。
- PostgreSQL + pgvector 相似度检索。
- DeepSeek `deepseek-v4-flash` 流式回答。
- 会话和消息历史、引用来源、模型调用记录、有用/无用反馈。
- 单用户、无登录，密钥仅由后端环境变量提供。

## 不在 MVP 范围内

- Agent、工作流编排、插件系统和任意工具调用。
- 多租户、注册登录、角色权限和计费。
- 消息队列、微服务或独立向量数据库。
- OCR、扫描 PDF、Office 文档和复杂表格恢复。

## 验收标准

- 新开发者可按照文档启动依赖、后端和前端。
- 可完成一次知识库问答并看到可核查的引用。
- 模型或文档处理失败时展示可理解的状态和错误。
- 可查看模型、耗时、Token、状态和用户反馈。
- API Key、私密文档和完整敏感请求不进入 Git 或默认日志。
- 固定评估问题能够重复运行并用于对比配置变化。
